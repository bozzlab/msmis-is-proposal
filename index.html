Development of a Prompt-Based AI Agent for Advertising Campaign Orchestration using Multi-Agent Architecture

แนวคิดหลักของโครงการ (Executive Summary)

โครงงานวิจัยนี้มุ่งพัฒนา Agent Harness (ระบบควบคุมเอเจนต์) บนโครงสร้าง Multi-Agent System (4 เอเจนต์ย่อย) โดยใช้เฟรมเวิร์ก LangGraph ร่วมกับสถาปัตยกรรม ReAct (Reasoning and Acting) เพื่อควบคุมและตั้งค่าโฆษณาบน Meta Ads API ผ่านภาษาธรรมชาติ ระบบผสานกลไก Human-in-the-Loop (HITL) เพื่อแสดงแผนงานให้ผู้ใช้ตรวจสอบก่อนยิงโฆษณาจริง รวมถึง Self-Healing Loop สำหรับแก้ไขข้อผิดพลาด API โดยอัตโนมัติ

1
ที่มาและความสำคัญของปัญหา
Background & Problem Statement
บริบทอุตสาหกรรม

แพลตฟอร์มโฆษณาดิจิทัลอย่าง Meta Ads Manager กลายเป็นโครงสร้างพื้นฐานของทีมการตลาดสมัยใหม่ อย่างไรก็ตาม การสร้างและจัดการแคมเปญต้องใช้ความเชี่ยวชาญด้านเทคนิคสูง ทั้งการนำทางผ่าน Dashboard ซับซ้อน การตั้งค่า Audience Targeting และการบริหารงบประมาณ ซึ่งสร้างภาระทางปัญญา (Cognitive Load) อย่างมากสำหรับนักการตลาดที่ไม่ใช่สาย Technical

ช่องว่างและปัญหา

มีช่องว่างสำคัญระหว่าง เจตนาทางการตลาด กับ การดำเนินการเชิงเทคนิค บนแพลตฟอร์มโฆษณา นักการตลาดใช้เวลา 25–40 นาทีต่อแคมเปญสำหรับการตั้งค่าด้วยมือ เครื่องมือปัจจุบันบังคับให้ผู้ใช้เรียน Interface เฉพาะแพลตฟอร์มแทนที่จะแสดงออกถึงเป้าหมายตามธรรมชาติ ความขัดแย้งนี้ลดประสิทธิภาพแคมเปญและเพิ่มระยะเวลา Time-to-Launch

วัตถุประสงค์การวิจัยและพัฒนา
Research & Development Objectives
01
ศึกษาและทดลองแนวทางการใช้ Agentic AI ในระบบนิเวศ MarTech และแสดงให้เห็นการประยุกต์ใช้จริงในสภาพแวดล้อม AdTech
02
ออกแบบและพัฒนา Multi-Agent System บน LangGraph ที่ตีความคำสั่งภาษาธรรมชาติและดำเนินการตั้งค่าแคมเปญโฆษณา Meta Ads โดยอัตโนมัติ
03
พัฒนากลไก Human-in-the-Loop (HITL) Verification Gate และ Self-Healing Loop ที่รองรับการ Approve/Reject แผนงานก่อน Launch และแก้ไข API Error อัตโนมัติ
04
พัฒนาระบบเพื่อตอบรับพฤติกรรมผู้ใช้ที่เปลี่ยนจากการค้นหาข้อมูล ไปสู่การสั่งงาน Agent แบบ Prompt-to-Action

Feature Scope
Detailed Feature Scope
F1: Conversational Chat Interface & Context ParsingUser Interface
รับคำสั่งภาษาธรรมชาติ ทำ Contextual Parsing แกะ Parameters เช่น: Budget · Objective · Age Range · Location · Image/Video (Creative) รองรับ Multi-turn Conversation และ Reconnect Logic
F2: Core System API & Campaign Reporting EngineSystem API
ระบบเซิร์ฟเวอร์หลังบ้าน (FastAPI) จัดการ User Authentication, Session Management และระบบ Campaign Report ดึงข้อมูลสถิติจากฐานข้อมูล PostgreSQL/Mongo มาประมวลผล
F3: State-Controlled Multi-Agent Orchestrator (LangGraph)Core Backend
กลไกควบคุม State Node ส่งต่องานระหว่าง 4 Agents มี Shared State Memory บันทึกประวัติการตัดสินใจของระบบทั้งหมดลง PostgreSQL
F4: HITL Verification Gate & Execution DashboardSafety Interface
UI พิเศษปรากฏเมื่อ Agent สรุปแผนสำเร็จ ผู้ใช้ตรวจสอบ Budget Structure, Creative Assets และกด Approve หรือ Reject + Feedback
F5: Meta Ads Platform Integration (API Bound)Integration Scope
รองรับ Traffic (เพิ่มยอดเข้าชมเว็บ) และ Conversion (ยอดขาย) Campaign Types + Daily Performance Report (Reach, Impressions, CPC, Amount Spent)
4
สถาปัตยกรรมระบบเอเจนต์
Multi-Agent Architecture — 4 Core Agents ภายใต้ LangGraph State Control
1. Orchestrator Agent

ทำหน้าที่ Supervisor รับ System Prompt หลัก แยกแยะ Intent ของผู้ใช้ ทำ Task Decomposition และควบคุมทิศทาง State Routing ให้เอเจนต์ตัวถัดไป

2. Strategy & Targeting Agent

รับผิดชอบโครงสร้างแคมเปญ การคำนวณงบประมาณ และ Audience Targeting แปลงคำสั่งพื้นที่ อายุ และพฤติกรรมให้เป็น API Parameter ที่ Meta ยอมรับ

3. Creative & Copywriting Agent

ใช้ Contextual Prompting วิเคราะห์บริบทของแบรนด์ สร้าง Headlines, Ad Copy และผูก Creative Media Assets ให้สอดรับกับพฤติกรรมกลุ่มเป้าหมาย

4. Guardrail & Validator Agent

ด่านสุดท้ายก่อนเรียก API ตรวจสอบ JSON Schema ตรวจงบประมาณไม่ให้เกิน Hard Ceiling Cap และตรวจสอบชิ้นงานตามนโยบายโฆษณา Meta

กลไกการทำงานเชิงเทคนิค
Technical Core — ReAct · HITL · Self-Healing · Guards
ReAct Pattern
วนลูป Thought → Action → Observation ผ่าน LangGraph State Machine จนกว่างานจะสำเร็จ ควบคุมผ่าน System Prompt ที่มี Role-Enforcement และ Contextual Framework
HITL Verification Gate
ระบบ State Interruption หยุดกราฟก่อน Launch แล้วแสดง Execution Plan บนหน้าจอ ผู้ใช้กด Approve เพื่อยิงแอด หรือ Reject + Feedback เพื่อให้ Agent ปรับแก้ใหม่
[+] Objective=TRAFFIC · Budget=5,000 THB/Day
[+] Target=Age 25-35 · Bangkok · Instagram
>>> AWAITING APPROVAL ···
Life Cycle Guards
Pre-tool: ตรวจ JSON Payload, Permission Token, Rate Limiting
Post-tool: ดักรับ API Response, ตรวจ Campaign/AdSet ID Status, บันทึก Trace ID
Self-Healing Loop
Auto-Correction: ถ้า JSON Error → Validator ส่ง Error Token ให้ Orchestrator แก้ไขและ Retry สูงสุด 2 ครั้ง
Fault Localization: ถ้าแก้ไม่ได้ ระบุจุดพังชัดเจนและแจ้ง User ผ่าน UI
